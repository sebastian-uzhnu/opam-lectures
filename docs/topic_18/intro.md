# Основні елементи управління

При розробці застосунків на базі Windows Forms використання вбудованих елементів керування (controls) є основою побудови користувацького інтерфейсу. Вони охоплюють базові задачі взаємодії: від виведення тексту до обробки команд та введення інформації.

## Label (Мітка)

Елемент `Label` використовується для відображення статичного тексту на формі, який користувач зазвичай не може редагувати безпосередньо.

- **Призначення:** Підписи до інших елементів управління (наприклад, "Введіть ім'я:" біля поля вводу), відображення статусних повідомлень.
- **Основні властивості:**
  - `Text` — власне текст, що відображається.
  - `AutoSize` — якщо `true`, розмір мітки автоматично підлаштовується під розмір тексту.
  - `Font` та `ForeColor` — налаштування шрифту та кольору тексту.

## TextBox (Текстове поле)

`TextBox` — це основний елемент для введення короткого тексту (одного або декількох рядків).

- **Призначення:** Введення користувачем даних (імена, числа, паролі).
- **Основні властивості:**
  - `Text` — містить введений текст.
  - `Multiline` — дозволяє вводити текст у декілька рядків, якщо встановлено `true`.
  - `PasswordChar` — символ (наприклад, `*`), який відображається замість введених символів (використовується для полів з паролем).
  - `ReadOnly` — забороняє зміну тексту користувачем.
- **Основні події:** `TextChanged` (спрацьовує при кожній зміні тексту).

## RichTextBox

Розширена версія звичайного текстового поля `TextBox`.

- **Призначення:** Форматований ввід та вивід тексту.
- **Основні властивості:**
  - На відміну від звичайного `TextBox`, дозволяє мати різні шрифти, кольори і розміри тексту в одному полі (властивість `SelectionFont`, `SelectionColor`).
  - `Rtf` — дозволяє отримувати чи задавати текст у форматі Rich Text Format.
  - Може містити зображення просто всередині тексту.

## Button (Кнопка)

Елемент `Button` — найпоширеніший спосіб запросити програму виконати певну дію.

- **Призначення:** Ініціація дії (кнопки "Зберегти", "Скасувати", "OK").
- **Основні властивості:**
  - `Text` — напис на кнопці.
  - `Enabled` — якщо `false`, кнопка стає сірою і на неї не можна натиснути.
  - `DialogResult` — визначає результат діалогу (наприклад, `OK` чи `Cancel`), якщо кнопка знаходиться на модальній формі.
- **Основна подія:** `Click`.

## CheckBox (Прапорець)

`CheckBox` дозволяє користувачеві вмикати або вимикати певну опцію (вибір типу "Так/Ні").

- **Призначення:** Вибір незалежних одна від одної опцій (наприклад, "запам'ятати мене", "додати розширену гарантію").
- **Основні властивості:**
  - `Checked` — логічне значення (`true` або `false`), що позначає стан прапорця.
  - `CheckState` — стан прапорця (може приймати значення `Checked`, `Unchecked`, або `Indeterminate` — стан часткового вибору, якщо властивість `ThreeState` дорівнює `true`).
- **Основна подія:** `CheckedChanged`.

## RadioButton (Перемикач)

Перемикачі `RadioButton` використовуються групами.

- **Призначення:** Вибір рівно однієї опції з декількох взаємовиключних (наприклад, вибір статі або типу доставки).
- **Основні властивості:**
  - `Checked` — `true`, якщо перемикач вибраний.
- **Особливість поведінки:** Всі елементи `RadioButton`, які знаходяться в одному батьківському контейнері (на формі безпосередньо або у Panel/GroupBox), об'єднуються в одну логічну групу. Якщо вибирається один `RadioButton`, всі інші в цій групі автоматично скидаються у стан `Checked = false`.
- **Основна подія:** `CheckedChanged`.
