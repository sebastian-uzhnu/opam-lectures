# Організація циклів у програмах.

Цикли в програмуванні дозволяють виконувати певний блок коду кілька разів підряд, поки виконується певна умова. В C# є
кілька основних типів циклів: `for`, `while`, та `do-while`, кожен з яких має свої особливості та застосування.

## Цикл `for`

Цикл `for`використовується тоді, коли кількість ітерацій відома заздалегідь. Він складається з трьох частин:
ініціалізації, умови виконання та оновлення лічильника.

### Синтаксис циклу `for`:

```csharp
csharp
Copy code
for (ініціалізація; умова; оновлення)
{
    // Тіло циклу
}

```

- **Ініціалізація**— це створення змінної-лічильника циклу.
- **Умова**— перевіряється перед кожною ітерацією; якщо умова хибна, цикл припиняється.
- **Оновлення**— змінює лічильник після кожної ітерації.

### Приклад циклу `for`:

```csharp
csharp
Copy code
for (int i = 0; i < 5; i++)
{
    Console.WriteLine("Ітерація: " + i);
}

```

У цьому прикладі змінна`i`ініціалізується значенням 0, цикл триває, поки`i`менше 5, і після кожної ітерації змінна`i`
збільшується на 1.

## Цикл `while`

Цикл `while` виконує блок коду доти, доки умова є істинною. Цей цикл застосовується, коли кількість ітерацій невідома
заздалегідь, і цикл залежить від певної умови.

### Синтаксис циклу`while`:

```csharp
csharp
Copy code
while (умова)
{
    // Тіло циклу
}

```

### Приклад циклу `while`:

```csharp
csharp
Copy code
int i = 0;

while (i < 5)
{
    Console.WriteLine("Ітерація: " + i);
    i++;
}

```

У цьому прикладі цикл виконується, поки значення змінної`i`менше 5. Після кожної ітерації змінна`i`збільшується на 1.

## Цикл `do-while`

Цикл `do-while` схожий на цикл `while` , але різниця в тому, що він спочатку виконує блок коду, а вже потім перевіряє
умову.
Таким чином, цей цикл виконається принаймні один раз, навіть якщо умова є хибною з самого початку.

### Синтаксис циклу `do-while`:

```csharp
csharp
Copy code
do
{
    // Тіло циклу
} while (умова);

```

### Приклад циклу `do-while`:

```csharp
int i = 0;

do
{
    Console.WriteLine("Ітерація: " + i);
    i++;
} while (i < 5);

```

У цьому прикладі цикл виконується, поки значення змінної `i` менше 5, але навіть якщо `i` дорівнювало б 5 з самого
початку,
цикл все одно виконав би першу ітерацію.

## Оператори `break` і `continue` в циклах

- Оператор `break ` дозволяє негайно припинити виконання циклу, незалежно від того, чи виконуються ще умови для
  продовження.
- Оператор `continue` пропускає поточну ітерацію і переходить до наступної.

### Приклад використання `break`:

```csharp
for (int i = 0; i < 10; i++)
{
    if (i == 5)
    {
        break; // Цикл припиниться, коли i досягне 5
    }
    Console.WriteLine(i);
}

```

### Приклад використання `continue`:

```csharp
for (int i = 0; i < 10; i++)
{
    if (i % 2 == 0)
    {
        continue; // Пропускаємо парні числа
    }
    Console.WriteLine(i);
}

```

## Вкладені цикли

У C# можна використовувати цикли всередині інших циклів. Це називається вкладеними циклами, і вони застосовуються для
вирішення завдань, що потребують багатовимірного перебору даних, наприклад, у роботі з матрицями або таблицями.

### Приклад вкладених циклів:

```csharp
for (int i = 0; i < 3; i++)
{
    for (int j = 0; j < 3; j++)
    {
        Console.WriteLine("i = " + i + ", j = " + j);
    }
}

```

У цьому прикладі зовнішній цикл управляє змінною `i`, а внутрішній — змінною `j`. Внутрішній цикл виконується повністю
для
кожного значення зовнішнього циклу.

## Цикл `foreach`

Цикл `foreach` використовується для ітерування елементів колекцій, масивів або списків. Він особливо зручний, коли вам
не
потрібно керувати індексами вручну.

### Синтаксис циклу `foreach`:

```csharp
foreach (тип змінна in колекція)
{
    // Тіло циклу
}

```

### Приклад циклу`foreach`:

```csharp
int[] numbers = { 1, 2, 3, 4, 5 };

foreach (int number in numbers)
{
    Console.WriteLine(number);
}

```

У цьому прикладі цикл `foreach` ітерується по масиву `numbers` і виводить кожен елемент на екран.

### Порівняння циклів:

| Цикл         | Коли використовувати                                  | Перевірка умови        | Мінімальна кількість виконань |
|--------------|-------------------------------------------------------|------------------------|-------------------------------|
| **for**      | Коли кількість ітерацій відома заздалегідь            | Перед кожною ітерацією | 0                             |
| **while**    | Коли кількість ітерацій невідома і залежить від умови | Перед кожною ітерацією | 0                             |
| **do-while** | Коли потрібно виконати хоча б одну ітерацію           | Після кожної ітерації  | 1                             |
