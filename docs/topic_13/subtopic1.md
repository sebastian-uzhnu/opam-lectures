# Режими та Точки зупинки

## Режими виконання

У Visual Studio існує два основних режими роботи при відлагодженні:

1.  **Неперервний (нормальний) режим (Running)**: Програма виконується без зупинок, поки не трапиться помилка або точка зупинки. Це звичайний стан роботи програми.
2.  **Режим з зупинками (Break Mode)**: Виконання програми призупинено. У цьому режимі ви можете переглядати значення змінних, стек викликів та покроково виконувати код.

## Виведення відлагоджувальної інформації

Для діагностики без зупинки програми часто використовують виведення інформації у вікно **Output**.

Клас `Debug` (з простору імен `System.Diagnostics`) дозволяє писати повідомлення, які видно лише у Debug-конфігурації:

```csharp
using System.Diagnostics;

Debug.WriteLine("Це повідомлення для відлагодження.");
Debug.WriteLineIf(x > 10, "x більше 10");
```

## Траспойнти (Tracepoints)

**Траспойнт (Tracepoint)** — це спеціальний вид точки зупинки, який **не зупиняє** виконання програми, а натомість виводить повідомлення у вікно Output. Це корисно для логування подій без переривання роботи програми.

**Як створити**:

1. Встановіть звичайний брейкпойнт (F9).
2. Натисніть на ньому правою кнопкою миші → **Actions** (Дії).
3. Введіть повідомлення, наприклад: `Current value of i is {i}`.
4. Галочка "Continue code execution" (Продовжити виконання) зазвичай встановлюється автоматично.

## Точки зупинки (Breakpoints)

**Брейкпойнт** — це маркер, який вказує відлагоджувачу зупинити виконання програми на певному рядку коду.

- **Вставка**: Натисніть F9 або клікніть на лівому полі редактора навпроти рядка коду.
- **Умовні брейкпойнти (Conditional Breakpoints)**: Дозволяють зупинятися лише тоді, коли виконується певна умова (Right Click → Conditions). Наприклад, `i == 5`.

Коли програма зупиняється на брейкпойнті, вона переходить у режим **Break Mode** (Пауза), і активний рядок підсвічується жовтим кольором.
