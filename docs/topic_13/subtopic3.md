# Розширені інструменти

Для складніших сценаріїв відлагодження Visual Studio надає додаткові інструменти.

## Вікно Immediate (Негайне виконання)

Вікно **Immediate Window** дозволяє виконувати код C# прямо під час відлагодження (у режимі Break Mode).

**Можливості**:

- **Обчислення виразів**: Можна перевірити результат виразу, наприклад, `? Math.Sqrt(x)` (знак питання `?` є скороченням для Print).
- **Зміна значень**: Ви можете змінити значення змінної на льоту, наприклад, `x = 50`. Це вплине на подальше виконання програми.
- **Виклик методів**: Можна вручну викликати методи для перевірки їх роботи, наприклад, `TestHelper.Validate(user)`.

## Вікно Command (Командне вікно)

Вікно **Command Window** використовується для виконання команд Visual Studio, а не коду програми. Наприклад, команди для збереження файлів, керування вікнами тощо. У контексті відлагодження воно використовується рідше, ніж Immediate Window, але може бути корисним для автоматизації середовища.

## Вікно Call Stack (Стек викликів)

Вікно **Call Stack** показує ланцюжок викликів методів, який привів до поточної точки виконання.

- Верхній рядок показує поточний метод (де знаходиться жовта стрілка).
- Наступний рядок — метод, який викликав поточний, і так далі до точки входу (`Main`).
- Подвійний клік на будь-якому рядку стека переміщує вас до коду відповідного методу (режим "зеленої стрілки"), дозволяючи переглянути значення змінних у тому контексті, хоча виконання фактично залишається в точці зупинки.

Це критично важливо для розуміння того, **як** програма опинилася в поточному стані.
