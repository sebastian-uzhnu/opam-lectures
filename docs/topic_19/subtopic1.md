# Власні діалогові вікна та інтеграція

Іноді стандартного `MessageBox` або діалогів для роботи з файлами недостатньо. Наприклад, вам потрібно запросити у користувача не просто підтвердження, а заповнити цілу форму (логін і пароль, дані для пошуку чи параметри налаштувань програми). Для цього у Windows Forms використовують **власні діалогові вікна**.

Будь-яка додаткова форма (спадкоємець класу `Form`), яку ви створюєте у своєму проекті, може виступати в ролі діалогового вікна.

## Створення власного діалогового вікна

Процес створення власного діалогу виглядає наступним чином:

1. Ви додаєте нову форму до проекту (наприклад, `SettingsForm`).
2. Розміщуєте на цій формі потрібні елементи керування (TextBox, CheckBox).
3. Розміщуєте як мінімум дві кнопки (зазвичай "ОК" та "Скасувати").
4. Налаштовуєте властивості форми так, щоб вона виглядала як діалогове вікно.

### Важливі властивості діалогової форми:

- `FormBorderStyle = FixedDialog` (щоб користувач не міг змінити розмір вікна).
- `MaximizeBox = false`, `MinimizeBox = false` (прибираємо кнопки згортання/розгортання).
- `StartPosition = CenterParent` (щоб діалог завжди з'являвся по центру батьківської форми, яка його викликала).

### Налаштування результату вікна (`DialogResult`)

Щоб форма працювала як класичний діалог (і її виклик повертав результат `DialogResult.OK` або `DialogResult.Cancel`), вам достатньо встановити властивість `DialogResult` для відповідних кнопок.
Наприклад, виділіть кнопку "ОК" (`btnOk`) у конструкторі, і у вікні властивостей знайдіть властивість `DialogResult`. Змініть її на `OK`. Для кнопки `btnCancel` змініть на `Cancel`.
Коли на ці кнопки буде натиснуто, форма **автоматично закриється** і поверне вибраний результат у батьківську форму. Ніякого додаткового коду для події `Click` писати не потрібно!

Також у самої форми є властивості:

- `AcceptButton`: кнопка, яка буде "натиснута" при натисканні користувачем клавіші `Enter`. Вкажіть тут `btnOk`.
- `CancelButton`: кнопка, яка буде "натиснута" при натисканні `Esc`. Вкажіть `btnCancel`.

## Два способи показу вікон: Show та ShowDialog

Існує принципова різниця в тому, як ви можете відобразити нову форму у Windows Forms.

### 1. Модальне вікно (`ShowDialog()`)

Модальне вікно "блокує" програму. Поки користувач не закриє цей діалог, він не зможе взаємодіяти з іншими (головними) вікнами вашої програми. Метод `ShowDialog()` призупиняє виконання коду у методі-викличнику, доки діалог не буде закрито.

Це ідеальний спосіб для вікон налаштувань, авторизації або попереджень.

```csharp
private void btnOpenSettings_Click(object sender, EventArgs e)
{
    // Створюємо об'єкт нашої форми налаштувань
    using (SettingsForm settingsForm = new SettingsForm())
    {
        // Показуємо форму модально і чекаємо результату
        if (settingsForm.ShowDialog() == DialogResult.OK)
        {
            // Якщо користувач натиснув ОК, читаємо нові дані з форми
            // (Про передачу даних поговоримо у наступному розділі)
            MessageBox.Show("Налаштування збережено!");
        }
        else
        {
            MessageBox.Show("Зміну налаштувань скасовано.");
        }
    }
}
```

### 2. Немодальне вікно (`Show()`)

Немодальне вікно дозволяє користувачеві вільно перемикатися між ним та іншими вікнами програми. Метод `Show()` просто відкриває вікно і одразу ж продовжує виконання наступного рядка коду, не чекаючи закриття форми.

Це зручно для плаваючих панелей інструментів (Tool Palettes), вікон з довідкою або інтерфейсів з багатьма однаковими незалежними вікнами.

```csharp
private void btnOpenHelp_Click(object sender, EventArgs e)
{
    HelpForm helpForm = new HelpForm();
    helpForm.Show(); // Форма відкрилась, але ми миттєво йдемо далі

    // Користувач може клікати і головну форму, і вікно HelpForm
}
```
