# Мультивіконні інтерфейси та MDI-додатки

Залежно від структури вікон, всі настільні застосунки (і не тільки Windows Forms) умовно поділяються на кілька архітектурних стилів:

1. **SDI (Single Document Interface)**: Програма має одне головне вікно, і кожен новий документ чи сутність відкривається у повністю новому, незалежному вікні ОС (приклад: класичний Блокнот (Notepad) у старих версіях Windows).
2. **MDI (Multiple Document Interface)**: Програма має одне велике батьківське вікно-контейнер, всередині якого відкриваються численні дочірні вікна. Всі дочірні вікна живуть виключно в межах батьківського вікна (приклад: старі версії Microsoft Excel чи Adobe Photoshop).
3. **Вікна з вкладками (TDI - Tabbed Document Interface)**: Сучасний підхід, де вікна чи документи відкриваються як вкладки (Tabs) (приклад: усі сучасні браузери, Visual Studio, сучасний Notepad++). Для реалізації у WinForms використовується елемент `TabControl`.

## Створення MDI-додатків у Windows Forms

Хоча MDI вважається дещо застарілим концептом на тлі переходу індустрії до вкладок (TDI), ця архітектура все ще широко використовується у різноманітних корпоративних додатках (бухгалтерія, системи управління базами даних).

У Windows Forms підтримка MDI вбудована на рівні базового класу `Form`.

### Крок 1: Створення батьківської форми (MDI Parent)

Щоб форма стала MDI-контейнером (обмеженим простором, в якому будуть відкриватись інші форми), достатньо встановити всього одну властивість:

- `IsMdiContainer = true`

Візуально після цього форма стане темно-сірою (колір робочої області MDI). Зазвичай до такої форми додають `MenuStrip` (Верхнє меню), щоб користувач міг створювати нові дочірні вікна чи управляти існуючими.

### Крок 2: Створення і показ дочірніх вікон (MDI Child)

Для того щоб нова форма відкрилася не як самостійне вікно десь на екрані, а як частина MDI-контейнера, перед викликом методу `Show()` потрібно встановити її властивість `MdiParent`.

Приклад коду для обробника меню `Файл -> Новий документ` у батьківській формі:

```csharp
private void newDocumentToolStripMenuItem_Click(object sender, EventArgs e)
{
    // Створюємо нову дочірню форму (наприклад, форму для редагування тексту)
    DocumentForm childForm = new DocumentForm();

    // Вказуємо, що поточна (головна) форма є її батьківським контейнером
    childForm.MdiParent = this;

    // Задаємо заголовок (для унікальності можна додавати лічильник)
    childForm.Text = "Документ " + (this.MdiChildren.Length);

    // Показуємо (обов'язково немодально!)
    childForm.Show();
}
```

### Керування дочірніми вікнами

У батьківській формі (`IsMdiContainer = true`) з'являється доступ до корисних властивостей:

- `MdiChildren`: Масив усіх відкритих на даний момент дочірніх форм. Дозволяє перебрати їх у циклі і, наприклад, зберегти дані з усіх відкритих документів відразу.
- `ActiveMdiChild`: Властивість, яка повертає ту дочірню форму, яка зараз знаходиться у фокусі (активна).

Крім того, батьківська форма має статичний метод `LayoutMdi`, який автоматично впорядковує відкриті вікна (каскадом, поруч один з одним по горизонталі чи вертикалі).

```csharp
private void cascadeMenu_Click(object sender, EventArgs e)
{
    // Розташувати всі вікна каскадом
    this.LayoutMdi(MdiLayout.Cascade);
}

private void tileHorizontalMenu_Click(object sender, EventArgs e)
{
    // Розташувати горизонтально
    this.LayoutMdi(MdiLayout.TileHorizontal);
}
```

MDI дозволяє створювати потужні інструменти для роботи з великою кількістю однотипних даних, даючи змогу легко перемикатись між ними на одному екрані, не засмічуючи при цьому панель завдань операційної системи (Taskbar) десятками окремих вікон.
