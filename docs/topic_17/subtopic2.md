# Властивості форм та модель подій

## Форми: загальні поняття та властивості

**Форма (Form)** у Windows Forms — це вікно або діалогове вікно, яке складає основу користувацького інтерфейсу програми. Усі форми є нащадками класу `System.Windows.Forms.Form`.

Форма сама по собі є контейнером, у який вкладаються інші елементи керування (controls). Вона має свій життєвий цикл та набір специфічних властивостей, методів і подій.

### Ключові властивості форм:

- **`Text`**: Заголовок вікна, який відображається у верхньому рядку.
- **`Name`**: Ім'я об'єкта форми в коді (використовується для звернення до неї з програмного коду).
- **`BackColor`, `ForeColor`**: Кольори фону та тексту за замовчуванням для елементів на формі.
- **`Size`, `Width`, `Height`**: Розміри вікна у пікселях.
- **`StartPosition`**: Визначає початкове положення вікна при запуску (наприклад, по центру екрану `CenterScreen`, або задається системою `WindowsDefaultLocation`).
- **`FormBorderStyle`**: Вигляд рамок вікна. Дозволяє заборонити зміну розміру вікна (`FixedSingle` або `FixedDialog`) або взагалі прибрати межі (`None`).
- **`WindowState`**: Початковий стан вікна (Згорнуте `Minimized`, Розгорнуте на весь екран `Maximized`, Звичайне `Normal`).
- **`MaximizeBox`, `MinimizeBox`**: Дозволяють приховати кнопки максимізації та мінімізації вікна у правому верхньому куті.
- **`TopMost`**: Якщо встановлено `true`, форма завжди буде знаходитися поверх усіх інших вікон у системі.
- **`Opacity`**: Непрозорість вікна (від 0.0 до 1.0).

## Модель подій у Windows Forms

Застосунки Windows Forms побудовані на основі **подієво-орієнтованої моделі**. Це означає, що потік виконання програми визначається подіями, які генеруються системою або користувачем. Додаток переважно знаходиться у стані очікування: він чекає, поки користувач клікне мишкою, натисне клавішу на клавіатурі або перетягне вікно.

Технічно події у Windows Forms базуються на стандартному механізмі **делегатів та подій у C#** (`event`).

### Як це працює?

1. **Джерело події (Publisher):** Елемент керування (наприклад, кнопка) має визначену подію (наприклад, `Click`).
2. **Обробник події (Subscriber / Event Handler):** Метод у вашому коді, сигнатура якого відповідає делегату події. Зазвичай обробники подій у WinForms приймають два параметри:
   - `object sender` — посилання на об'єкт, який ініціював подію (хто натиснув кнопку). Це корисно, якщо один обробник прив'язаний до кількох елементів.
   - `EventArgs e` (або його нащадки) — об'єкт, що містить додаткову інформацію про подію (наприклад, координати миші у `MouseEventArgs` або яка клавіша натиснута у `KeyEventArgs`).
3. **Підписка:** Встановлюється зв'язок між подією та обробником (`button1.Click += new EventHandler(button1_Click);`). Цей код зазвичай генерується автоматично у файлі `.Designer.cs`.

### Найважливіші події форми:

- **Події життєвого циклу:**
  - `Load`: Відбувається перед першим відображенням форми. Ідеальне місце для ініціалізації даних, завантаження налаштувань.
  - `Shown`: Відбувається після того, як форма стала видимою вперше.
  - `FormClosing`: Викликається перед закриттям форми. Дозволяє скасувати закриття (наприклад, запитати користувача "Ви впевнені, що хочете вийти?").
  - `FormClosed`: Викликається після закриття форми перед тим, як звільняються ресурси.

- **Події взаємодії:**
  - `Click`, `DoubleClick`: Клік мишею по формі чи елементу.
  - `KeyDown`, `KeyUp`, `KeyPress`: Натискання клавіш клавіатури, коли форма або її елемент має фокус вводу.
  - `MouseEnter`, `MouseLeave`, `MouseMove`: Рух курсору миші в межах форми.
  - `Resize`: Зміна користувачем розмірів вікна.

Розуміння того, в якому порядку відбуваються події (особливо події життєвого циклу: `Load` -> `Shown` -> ... -> `FormClosing` -> `FormClosed`) є критично важливим для правильної розробки застосунків.
