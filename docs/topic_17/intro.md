# «Традиційна» модель програмування та Windows Forms

## «Традиційна» модель програмування на платформі .NET

Історично склалося так, що розробка застосунків з графічним інтерфейсом користувача (GUI) для операційної системи Windows базувалася на використанні Windows API (WinAPI). Цей підхід вимагав написання великої кількості низькорівневого коду для створення вікон, обробки повідомлень ОС та малювання елементів на екрані.

З появою платформи .NET Framework у 2002 році компанія Microsoft запропонувала нову, об'єктно-орієнтовану модель програмування настільних додатків — технологію **Windows Forms**. Вона стала «традиційною» моделлю розробки для .NET, сховавши складність викликів WinAPI за зручним набором керованих класів (managed classes).

Основна ідея традиційної моделі (на відміну від пізнішої WPF чи сучасних веб-технологій) полягає в тісній інтеграції з базовими механізмами Windows (GDI+ для малювання, User32 для керування вікнами).

## Технологія Windows Forms

**Windows Forms (WinForms)** — це платформа розробки графічного інтерфейсу користувача з відкритим вихідним кодом (нині частина .NET Core / .NET 5+), яка дозволяє створювати насичені настільні клієнтські застосунки для Windows.

### Ключові особливості Windows Forms:

- **Швидка розробка (RAD - Rapid Application Development):** Завдяки візуальному конструктору (Designer) у Visual Studio, розробники можуть створювати інтерфейси перетягуванням елементів керування (drag-and-drop).
- **Багата бібліотека елементів керування:** Наявність готових кнопок, текстових полів, списків, таблиць (DataGrid), меню тощо.
- **Подієво-орієнтована архітектура:** Логіка програми будується навколо реакції на події (натискання кнопок, рух миші, введення тексту).
- **Обгортка над Win32 API:** Класи у просторі імен `System.Windows.Forms` фактично є обгортками над нативними компонентами Windows.

## Загальна структура додатку з графічним інтерфейсом

Будь-який додаток Windows Forms має певну стандартну структуру:

1. **Точка входу (Main method):** Зазвичай знаходиться у файлі `Program.cs`. Це статичний метод `Main()`, з якого починається виконання програми.
2. **Цикл повідомлень (Message Loop):** Метод `Application.Run()` запускає стандартний цикл повідомлень застосунку, який безперервно очікує та обробляє події операційної системи.
3. **Головна форма:** Це основне вікно програми (переважно клас `Form1`), яке передається у метод `Application.Run(new Form1())`.
4. **Файли форми:**
   - `Form1.cs` — файл, де розробник пише логіку (обробники подій, методи).
   - `Form1.Designer.cs` — файл, що автоматично генерується середовищем розробки. У ньому міститься код ініціалізації всіх елементів керування, їх розміщення та налаштування властивостей.
   - `Form1.resx` — файл ресурсів для збереження специфічних для форми об'єктів (наприклад, зображень, локалізованих рядків).
