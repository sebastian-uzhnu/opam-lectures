# Створення першого застосунку

## Створення першого проекту

Давайте розглянемо кроки для створення найпростішого додатку Windows Forms у Visual Studio, який буде запитувати ім'я користувача і вітати його.

1. **Створення проекту:**
   - Відкрийте Visual Studio та виберіть **"Create a new project"** (Створити новий проект).
   - Знайдіть та виберіть шаблон **"Windows Forms App"** (зверніть увагу на версію платформи, бажано вибирати варіанти на базі .NET Core/.NET 5+, а не застарілий .NET Framework, якщо це можливо). Натисніть "Next".
   - Введіть назву проекту (наприклад, `HelloWinFormsApp`), виберіть розташування і натисніть "Create".

2. **Знайомство з формою:**
   - Після створення проекту перед вами з'явиться порожня форма `Form1` у вікні дизайнера.
   - У вікні **Properties** змініть властивість `Text` форми на "Моя перша програма". Заголовок вікна одразу зміниться.
   - Змініть `StartPosition` на `CenterScreen`, щоб вікно завжди з'являлося по центру дисплея.

3. **Додавання елементів керування:**
   - Відкрийте **Toolbox** (Панель елементів).
   - Перетягніть на форму такі елементи:
     - `Label` (Мітка) — для підказки користувачеві.
     - `TextBox` (Текстове поле) — для введення імені.
     - `Button` (Кнопка) — для дій користувача.
   - Виділіть доданий `Label`. У вікні властивостей змініть його `Text` на "Введіть ваше ім'я:".
   - Виділіть `TextBox`. Змініть його властивість `Name` на `txtName`.
   - Виділіть `Button`. Змініть `Name` на `btnGreet`, а `Text` на "Привітатися".

## Прості практичні приклади взаємодії

Тепер додамо логіку нашій програмі. Ми хочемо, щоб при натисканні на кнопку програма зчитувала текст із текстового поля і виводила віконце з привітанням.

### Додавання обробника події

Двічі клацніть лівою кнопкою миші по кнопці "Привітатися" на формі. Visual Studio автоматично згенерує метод-обробник події `Click` у коді файлу `Form1.cs` і перемкне вас туди.

```csharp
using System;
using System.Windows.Forms;

namespace HelloWinFormsApp
{
    public partial class Form1 : Form
    {
        public Form1()
        {
            InitializeComponent();
        }

        // Автоматично згенерований обробник події кліку по кнопці
        private void btnGreet_Click(object sender, EventArgs e)
        {
            // 1. Отримуємо текст, введений користувачем
            string userName = txtName.Text;

            // 2. Перевіряємо, чи користувач нічого не ввів
            if (string.IsNullOrWhiteSpace(userName))
            {
                // Виводимо повідомлення про помилку
                MessageBox.Show("Будь ласка, введіть своє ім'я!", "Помилка", MessageBoxButtons.OK, MessageBoxIcon.Warning);
            }
            else
            {
                // 3. Формуємо привітання та показуємо його
                string message = $"Привіт, {userName}! Вітаємо у світі Windows Forms!";
                MessageBox.Show(message, "Привітання", MessageBoxButtons.OK, MessageBoxIcon.Information);
            }
        }
    }
}
```

### Запуск програми

Натисніть кнопку **Start** (або F5) у Visual Studio.

1. З'явиться ваше вікно.
2. Якщо ви натиснете кнопку "Привітатися", нічого не ввівши, з'явиться стандартне діалогове вікно попередження (`MessageBox`) із повідомленням "Будь ласка, введіть своє ім'я!".
3. Введіть ім'я (наприклад, "Іван") і натисніть кнопку. Ви побачите віконце з текстом "Привіт, Іван! Вітаємо у світі Windows Forms!".

### Аналіз взаємодії

Цей простий приклад демонструє кілька концептуальних речей "Традиційної" моделі:

- **Читання властивостей:** Рядок `txtName.Text` зчитує поточний вміст текстового поля, яке користувач міг змінити з клавіатури.
- **Стандартні діалоги:** Клас `MessageBox` часто використовується у WinForms для виведення простих повідомлень та отримання швидких відповідей від користувача (наприклад, кнопки "Так"/"Ні").
- **Реакція на події в реальному часі:** Весь код виконується виключно як реакція на подію `Click`. Якщо кнопку не натискати, програма просто чекає в головному циклі повідомлень, споживаючи мінімум ресурсів процесора.

Щоб розширити програму, ви можете додати обробник події `TextChanged` для `txtName`, який, наприклад, буде робити кнопку неактивною (`btnGreet.Enabled = false`), поки поле пусте. Це робить інтерфейси більш "живими" та чуйними на дії користувача.
